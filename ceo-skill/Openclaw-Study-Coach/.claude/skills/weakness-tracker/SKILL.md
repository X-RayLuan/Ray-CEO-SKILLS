# 纵向画像 Skill

## 这个 skill 是干什么的

让 agent **每次新对话都"记得"孩子**——不是技术上的记忆,是通过三个文件把观察沉淀下来,下次启动时 CLAUDE.md 自动加载这三份当上下文。

三个文件:

| 文件 | 内容 | 谁能看 |
|---|---|---|
| `data/profile/weakness-profile.md` | 反复出现的卡点、需要持续训练的方向 | 仅家长 + agent |
| `data/profile/strengths.md` | 她做对了什么、哪些已经稳了、可以拿来鼓励 | 孩子 + 家长 + agent |
| `data/profile/interests.md` | 她喜欢什么(用来包装练习、做联想类引导) | 孩子 + 家长 + agent |

## 什么时候触发

### 1. 即时触发(reading-comp-tutor / writing-coach / mistake-tracker 内部)

每次有具体观察(她在某类题上卡 / 答出 / 提到她喜欢什么),**追加一行**到对应文件。这是**累加层**,不做提炼。

### 2. 收尾触发(daily-report 结尾)

daily-report 写完日报后,扫一遍当天对话,把**当天关键观察**抽 1-3 条追加到三个文件。

### 3. 综合触发(weekly-report 结尾,每周一次)

weekly-report 跑完后,**重写**(不是追加)weakness-profile.md,做提炼:
- 删掉 1-2 次的偶发观察
- 把多次出现的提炼成 pattern
- 保留 ≤ 7 条核心 pattern,按重要度排序

strengths.md 也做同样的整理。interests.md 只做去重,不删(兴趣是积累的)。

## 文件格式

### weakness-profile.md

```markdown
# 弱项画像 — 小妹(meimei)

> 这是 agent 用来"记得她"的核心文件之一。
> 每次新对话开始,agent 会读这份,知道她最近在哪些方向需要持续训练。
> ⚠️ 这份文件**孩子看不到原版**,只给家长 + agent。

最后更新: 2026-05-22 (week 2026-W21)

## 当前关注 pattern(≤ 7 条,按重要度)

### 1. 阅读理解 - 代入类发散题(⚠️ 高频)
- 表现:"如果你是 X 你会怎么做"类题答不出
- 出现频次:本周 5 次,上周 3 次
- 已尝试方案:给 3 个方向选项后能答(给"跟人物一样 / 不一样 / 第三种办法"作为方向)
- 下一步:连续给 7 天代入类小练习,每天 1 道,逐步减少选项辅助

### 2. 作文 - 不会开头 / 害怕动笔
- 表现:看到作文题立刻说"我不会";写第一句要 5 分钟
- 已尝试方案:不让她写开头,先聊 + 先写中间一段最有画面的
- 效果:有缓解,周一周三都写出第一段了
- 下一步:再坚持 2 周这个流程,然后试一次"先写开头"看是否改善

### 3. ...

## 已观察但暂不归 pattern(出现 ≤ 2 次,继续观察)

- 数学:应用题审题偶尔漏条件(2 次,继续看)
- 语文:形近字偶尔写错(1 次,不归)

## 历史已解决(给 agent 知道:不要再练了)

- 2026-W19:阅读理解的"找证据"类题已经稳了,不再主动出
```

### strengths.md

```markdown
# 强项画像 — 小妹

> 这份文件**孩子也能看**(她要求看自己强项时)。
> 每次新对话开始,agent 读这份,做反馈时优先调用她的强项。

最后更新: 2026-05-22

## 已稳的能力(可以放心用 / 可以拿来鼓励)

- ✓ 阅读理解 - 找证据类:能直接圈关键词
- ✓ 数学 - 口算速度:60 题以内不出错
- ✓ 英语 - 单词造句:能用新词写出有画面的句子

## 最近做得好的具体瞬间(给孩子看的版本)

- 2026-05-22: 在《小猫》那篇里,你找到了"轻轻地"作为答案的证据 ⭐
- 2026-05-21: 你作文里写"妈妈的裙子是蓝色的"——颜色细节用得很好 ⭐
- 2026-05-20: 数学应用题第 3 题,你自己发现了"多 5 个"应该用加法 ⭐

## 性格特征(在反馈时利用)

- 喜欢被指出**具体哪里做对**,不喜欢空夸
- 对"她有进步"的反馈比"她是好孩子"的反馈反应大
- 喜欢闯关式叙事(用"关"比用"题"她更投入)
```

### interests.md

```markdown
# 兴趣画像 — 小妹

> 这份文件用来**包装练习题**(选阅读材料、出作文题、做联想引导)。
> 孩子可以看(她可能会主动告诉 agent "我还喜欢 XX",这时就追加)。

最后更新: 2026-05-22

## 喜欢的主题(agent 出题时优先用)

- 小动物(尤其是猫)
- 画画
- (etc... 持续收集)

## 喜欢的故事 / 书 / 动画(可以拿来做阅读理解材料)

- (待收集)

## 反感的题型 / 元素

- 不喜欢战争 / 打仗类故事(2026-05-22 提到过)
- 不喜欢被纠正"错"字眼,改成"我们再看一眼"

## 日常生活线索(可以做联想类引导的素材)

- 她家有 [宠物 / 植物 / 等]?(待收集)
- 周末通常做什么?(待收集)
- 最近读了什么书?(待收集)
```

## 即时追加的格式(reading-comp-tutor / writing-coach 调用)

不要写完整段。**追加单行**到 weakness-profile 最后,格式:

```markdown
- YYYY-MM-DD: [观察描述,一句话] ([题目/学科])
```

例:
```markdown
- 2026-05-22: 代入类题答不出,给方向选项后能答(阅读理解《小猫》)
- 2026-05-22: 作文里第一次主动用了感受词"忐忑"(作文《那一天》)
```

strengths.md 追加同样格式,但写**具体做对的事**。

interests.md 追加格式:

```markdown
- YYYY-MM-DD: 提到喜欢 [XX],在 [对话上下文] 中
```

## 周综合(weekly-report 调用)

weekly-report 跑完后,你**重写**(不是追加)weakness-profile.md。流程:

1. 读当前 weakness-profile.md 的所有追加观察(包括"当前关注 pattern" + "已观察但暂不归 pattern")
2. 读本周 trends.csv 的 7 行
3. 读本周 daily reports
4. 提炼:
   - 同一类卡点出现 ≥ 3 次 → 升级到"当前关注 pattern"
   - 当前关注里的 pattern,如果本周没再出现 + 上周也没 → 移到"历史已解决"
   - "已观察但暂不归 pattern" 里的,如果本周再没出现 → 删
5. 给每个当前 pattern 加 4 个字段:**表现 / 频次 / 已尝试方案 / 下一步**
6. 按重要度排序,保留 ≤ 7 条
7. 写完后告诉用户:"weakness-profile 已重写,本周新增 X 条,移除 Y 条,合并 Z 条"

strengths.md 同样处理。但"已稳的能力"是只增不减的(她证明过会的事,不要删)。"最近做得好的具体瞬间"保留最近 14 天,旧的归档到 `data/profile/strengths_archive/`。

## 怎么把 weakness-profile 用起来

agent 在以下场景**主动读** weakness-profile:

1. **开始一次新对话时**(CLAUDE.md @import 自动加载)
2. **决定今天出什么练习时**(reading-comp-tutor / writing-coach 调用)
3. **生成日报"明日 1 个重点"时**(daily-report 优先选 pattern 里的方向)
4. **生成周报"下周 1 个重点"时**(weekly-report 选最高优 pattern)

agent 不主动跟孩子说"你的弱项是 XXX"。**永远不说**。但 agent 出的练习、给的反馈、引导的方向,会被这份画像驱动。

## 给孩子的"成就感面板"(strengths 的孩子可见版本)

孩子如果说"我哪里厉害?"、"我有进步吗?",你给一个**精简版**:

> 我让你看看你的成就墙 ⭐
>
> **已经稳的事情**:
> - 阅读理解找证据 ✓
> - 数学口算 ✓
> - 英语造句 ✓
>
> **最近这一周你做得特别厉害的**:
> - 5/22:在《小猫》那篇里找到"轻轻地"做证据
> - 5/21:作文里用了"蓝色的裙子"——颜色细节
> - 5/20:数学应用题自己发现"多 5 个"该用加法
>
> 你看,这些都是你自己做到的 ✨

**不**给她看 weakness-profile 内容。

## 硬规则

1. **即时追加只写 1 句**,不写段落。这是"日志层"。
2. **周综合时才提炼**。中间不要乱改。
3. **永远不让孩子看 weakness-profile 原版**。她要求看 → 给 strengths 的精简版。
4. **pattern 里必须有"已尝试方案" + "下一步"**。否则画像就是抱怨,不是行动。
5. **interests 只增不减**。除非她明确说"我不喜欢 XX 了"。
6. **每次写完三个文件,在对话里告诉用户更新了什么**(让家长能感觉到系统在变聪明)。
